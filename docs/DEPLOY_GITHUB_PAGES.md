# GitHub Pages 배포 메모

## 방식 1: GitHub Actions 워크플로 사용
이 패키지에는 `.github/workflows/pages.yml`이 포함되어 있습니다.
파일을 그대로 업로드하고 `main` 브랜치에 푸시하면 배포 워크플로가 동작하도록 구성했습니다.

## 방식 2: 브랜치 기반 배포
정적 파일만 단순 배포할 경우 GitHub 저장소의 **Settings → Pages**에서
브랜치와 폴더를 직접 선택해도 됩니다.

## 운영 팁
- 공개 저장소에는 실데이터를 올리지 않습니다.
- 첫 배포 후 브라우저 캐시 때문에 이전 서비스 워커가 남을 수 있습니다.
  이 경우 새로고침(Ctrl+F5) 또는 사이트 데이터 삭제 후 다시 확인하세요.
- PWA 설치는 브라우저별 지원 상태가 다를 수 있습니다.
